# Digital Forensics Analysis – Planet's Prestige

## Overview
This challenge involved analyzing provided artifacts to identify hidden or non-obvious information through structured forensic techniques.

## Objective
- Analyze given files and extract meaningful data
- Identify anomalies and hidden structures within the dataset

---

## Initial Assessment
- The provided data contained multiple embedded elements
- Some files appeared normal at first glance but required deeper inspection

---

## Analytical Approach

### 1. File Validation
- Verified actual file types instead of relying on extensions
- Identified discrepancies between file extension and true format

### 2. Structural Analysis
- Decomposed container files (e.g., archives)
- Inspected internal file structures and relationships

### 3. Data Inspection
- Extracted readable strings and metadata
- Investigated embedded or encoded content

### 4. Iterative Analysis
- Repeated validation and extraction across multiple layers
- Identified hidden or non-obvious artifacts through layered inspection

---

## Key Findings
- Detected inconsistencies in file structure and representation
- Identified embedded data requiring further extraction
- Successfully uncovered hidden information through multi-step analysis

---

## Analyst Notes
- File extensions should never be trusted during forensic analysis
- Multi-layered data structures are commonly used to obscure content
- Iterative inspection is essential when dealing with obfuscated artifacts

---

## Takeaways
- Always validate file types using reliable methods
- Break down complex artifacts into smaller components
- Maintain a structured approach to avoid missing hidden data
